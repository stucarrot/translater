# 자막·문서 번역기 개선 작업 인수인계서 (v7)

## 0. 한 줄 요약
이전 대화(v6)까지 쌓아온 번역 엔진(마커 정렬, 포맷별 프로파일, FB2 이미지/구조 보존, 서술체 실험 옵션 등)은 전혀 손대지 않았고, **이번 대화(v7)에서는 사용자 요청으로 '단어 목록(단어장)' 추출 기능 전체를 코드베이스에서 완전히 제거**했음. 단어장 엔진(레벨 판정·사전 조회·고유명사 필터·전체개수 모드·인라인 삽입), Anki `.apkg` 내보내기, 단어장 파일(.txt) 내보내기, 관련 UI(슬롯 카드의 "📘 단어 학습 목록 추가" 박스), 각 파서 `rebuild()`의 임베드 매개변수(`embedMap`/`embedMode`)까지 전부 들어냈고, **번역 핵심 로직·글로서리(용어 고정)·FB2/EPUB 구조 보존 기능은 그대로 남겨뒀음**. 코드 정적 검사(`node --check`)와 식별자 잔존 여부 grep까지는 마쳤지만, **제거 후 실제 브라우저에서 각 포맷을 번역해보는 실사용 회귀 테스트는 이번 대화에서 못 함 — 다음 세션 최우선.**

## 1. 배경 / 목표
- v4~v6에서 만들었던 단어장 시스템(레벨별 추출, 고유명사 필터링, 전체개수 모드, 인라인 `[뜻]` 삽입, Anki `.apkg`/텍스트 파일 내보내기)을 사용자가 더 이상 원하지 않아 **전면 삭제** 요청
- 목표는 두 가지: (1) 단어장 관련 코드·UI·데이터를 남김없이 제거, (2) 그 과정에서 번역 엔진·글로서리·FB2/EPUB 처리 등 **다른 기능은 절대 망가뜨리지 않기**
- 참고: 이 작업은 순수 "제거" 리팩터링이라 품질/속도 개선은 대상이 아니었음. 4~6장에서 설명하는 번역 알고리즘 자체는 v6 상태 그대로임

## 2. 파일 위치
- 작업 파일: `index.html` (단일 파일 웹앱, 순수 JS/HTML/CSS)
- **대화 첨부파일 중 가장 마지막 것이 최신 코드입니다.**

## 3. 이번 대화(v7)에서 제거한 것 — 위치별 정리

### 3-1. 데이터 / 엔진 (JS)
| 제거 대상 | 설명 |
|---|---|
| `VOCAB_BANDS` | 영어/스페인어/러시아어 CEFR 근사 레벨용 대용량 빈도 단어 리스트. 자체 `<script>` 블록 통째로 삭제 |
| `SUPPORTED_VOCAB_LANGS`, `LEVEL_ORDER`, `POS_KR` | 단어장 지원 언어, 레벨 순서, 품사 한글 라벨 상수 |

| `tokenizeByLang`, `isCapitalizedToken`, `escapeRegExpLiteral` | 언어별 토큰화 + 고유명사(대문자) 판별 + 정규식 이스케이프 헬퍼 |
| `applyInlineGloss` | 본문 단어 옆에 `[뜻]`을 삽입하던 함수 |
| `_dictCache`, `gdictLookup` | 구글 사전(`dt=bd`) 조회 + 캐시 |

| `<script src=".../sql-wasm.js">` | `buildAnkiApkg` 전용이었던 sql.js(WASM SQLite) CDN 로더. 다른 곳에서 안 쓰여서 같이 제거 |

### 3-2. 상태 / UI (JS + HTML)
- `newSlot()`의 `vocabEnabled`, `vocabLevel`, `vocabMaxWords`, `vocabExtractMode`, `vocabTotalWords`, `vocabDisplayMode`, `vocabEmbed`, `vocabExportFile`, `vocabExportExamples`, `vocabExportAnki` 필드 전부 삭제 (`glossary`는 유지)
- `renderSlotCards()`의 슬롯 카드에서 "📘 단어 학습 목록 추가" 박스(레벨 선택, 추출모드 드롭다운, 개수 입력, 표시방식 선택, 임베드/파일생성/예문/Anki 체크박스, 안내문구)와 관련 이벤트 리스너 전부 삭제. **"🔒 고유명사·용어 고정"(글로서리) 박스는 그대로 유지** — 같은 `.vocab-box` CSS 클래스를 재사용하고 있었을 뿐 단어장과는 무관한 별개 기능이라 손대지 않음
- `renderFileBlockList()`의 파일 카드 칩에서 단어장 표시(📘 레벨+개수) 제거, 언어 칩만 남김
- "전체 파일에 적용" 버튼의 안내 문구에서 "단어장" 언급 제거(용어고정만 언급하도록 수정)
- 하단 실행 패널의 안내 문구(`.note`)에서 "단어장 기능은 영어·스페인어·러시아어일 때 지원..." 문장 제거
- 더 이상 쓰이지 않게 된 `.vocab-chip`, `.vocab-level-row` 계열 CSS 삭제

### 3-3. 파서 `rebuild()` 시그니처 (SRT/VTT/TXT/EPUB/FB2)
- v6에서 다섯 파서 모두 `rebuild(translations, position, embedMap, embedMode[, format])`로 확장돼 있던 것을, 단어장 임베드 자체가 없어졌으므로 **`rebuild(translations, position)`** (FB2만 `format` 인자 추가로 `rebuild(translations, position, format)`)로 되돌림
- FB2의 `renderNodesToXhtmlParts`, `rebuildFb2Passthrough`에서도 `embedMap`/`embedMode` 분기(인라인 삽입, 문단 아래 단어장 `<p class="vocab">` 삽입)를 전부 제거하고, 원문/번역문 교체·위/아래 배치 로직만 남김
- EPUB `rebuild()`에서 문단 아래 `vocab-block` `<div>` 삽입 로직 제거

### 3-4. 메인 실행 파이프라인 (`runTranslation`)
- 슬롯별 번역 후 `buildVocabMap` 호출, `vocabMap`/`vocabInlineMap`/`vocabList` 구성, `effectiveVocabDisplay`(교체 위치일 때 인라인→목록 강제 전환) 계산, `embedVocabMap` 결정 로직을 통째로 삭제
- `parsed.rebuild(...)` 호출에서 단어장 관련 인자 제거 → `parsed.rebuild(translations, slot.position, file.fb2OutputFormat || 'epub')`
- 단어장 텍스트 파일(`_단어장.txt`) 생성/zip 추가/로그 출력 블록 삭제
- Anki `.apkg` 생성(`buildAnkiApkg` 호출)/zip 추가/로그 출력 블록 삭제
- `translateBlocks`의 반환값에서 더 이상 안 쓰는 `detectedSource`를 구조분해에서 제외(단어장 자동감지 언어 판별에만 쓰였음). **`translateBlocks` 함수 자체는 그대로 두 값을 반환하며, 호출부에서 `translations`만 취하도록만 바꿈** — 함수 시그니처/내부 로직은 무손상

## 4. 제거 과정에서 특별히 조심한 것 (다른 기능 보호)
1. **글로서리(용어 고정) 기능과의 CSS 클래스 충돌**: 단어장 박스와 글로서리 박스가 똑같이 `.vocab-box`/`.vocab-toprow`/`.vocab-note` CSS 클래스를 재사용하고 있었음(원래 그냥 "박스 스타일" 이름이었을 뿐, 단어장 전용이 아니었음). 클래스 자체를 지우면 글로서리 박스 스타일이 깨지므로 **CSS 클래스는 그대로 두고, 단어장 쪽 HTML 마크업/로직만 골라서 제거**함
2. **`rebuild()` 시그니처 변경이 호출부와 어긋나지 않는지 확인**: 5개 파서 전부 새 시그니처로 통일했고, 메인 파이프라인의 단일 호출부(`parsed.rebuild(...)`)도 그에 맞춰 인자 개수를 맞춤. SRT/VTT/TXT/EPUB는 `format` 인자를 아예 안 받고 무시하며, FB2만 3번째 인자로 받아 씀 — JS는 여분 인자를 그냥 무시하므로 호출부 하나로 다섯 파서 모두 안전하게 커버됨
3. **`escapeRegExpLiteral`, `tokenizeByLang` 등 이름이 범용적으로 보이는 헬퍼가 다른 기능(글로서리 치환, 마커 정렬 등)에서도 쓰이는지 사전에 grep으로 전수 확인** 후 삭제 — 전부 단어장 전용이었고 다른 곳에서 재사용되지 않음을 확인하고 제거함
4. **`sql.js` CDN 스크립트 태그 제거 전에 JSZip과 혼동하지 않도록 확인**: JSZip은 EPUB 파싱/빌드에도 쓰이므로 유지, sql.js는 Anki apkg 생성에만 쓰여서 그것만 제거
5. 제거 후 **`node --check`로 인라인 JS 전체 구문 검사 통과**, `vocab`/`VOCAB`/`gdictLookup`/`applyInlineGloss`/`buildAnkiApkg` 등 식별자가 코드에 전혀 남아있지 않은지 grep으로 재확인함

## 5. 검증 상태 (⚠️ 중요 — 다음 세션 최우선)
- **정적 검사만 완료**: JS 구문 오류 없음(`node --check`), 삭제 대상 식별자 잔존 없음, `rebuild()` 시그니처와 호출부 인자 개수 일치 확인
- **브라우저 실사용 테스트는 이번 대화에서 전혀 못 함**. 특히 다음을 다음 세션에서 반드시 확인할 것:
  1. SRT/VTT/TXT/EPUB/FB2 각 포맷을 실제로 업로드해서 번역이 여전히 정상 동작하는지 (단어장 제거가 번역 자체에 영향 없어야 함)
  2. 글로서리(🔒 고유명사·용어 고정) 입력창이 슬롯 카드에서 정상적으로 보이고 동작하는지 (단어장 박스 제거 후 레이아웃이 깨지지 않았는지)
  3. FB2 파일을 "EPUB로 변환" / "FB2로 유지" 둘 다로 내보내서 v6에서 확인했던 표지·인라인삽화·다단계 목차가 여전히 정상인지 (rebuild 시그니처를 손댔으므로 회귀 여부 재확인 필요)
  4. 파일 목록/슬롯 카드 UI에 단어장 관련 잔재(빈 박스, 깨진 스타일 등)가 화면상 없는지 육안 확인
  5. "이 옵션 전체 파일에 적용" 버튼이 여전히 정상 동작하는지(글로서리/언어/위치 설정만 복사하면 됨)

## 6. v6까지의 알려진 한계 (이번 대화에서 다루지 않음, 그대로 유효)
아래 항목들은 이번 대화의 작업 범위(단어장 제거)와 무관하며 v6 인수인계서 내용이 그대로 유효함:
- 문서 번역에서 주어(대명사) 생략 문제 (미해결, `contextBefore=0` 실험 여지 있음)
- '합니다체' 번역 경향 (부분 대응만 있음, 현재형 동사 활용 처리는 한계)
- FB2 EPUB 변환 결과물의 실사용(캘리버 등) 검증은 v6에서도 안 됐고 이번 대화에서도 추가 검증 없음
- 문장이 여러 자막 큐에 걸치는 경우의 어색한 분할, 자동 용어 일관성 미구현 등은 동일하게 남아있음
- v4~v6 변경분(포맷 프로파일, 고급옵션, Anki — 단, Anki는 이번에 기능 자체가 삭제됐으므로 더 이상 검증 대상 아님, FB2 구조 보존)의 실사용 미검증 이슈도 단어장 관련 항목만 자연 소멸했을 뿐 나머지는 그대로 남음
